# SEO 优化指南

本指南帮助你的内容更快被搜索引擎收录。

---

## 第一步：部署后配置

部署完成后，在开始写文章之前，先完成以下配置。

### 后台 SEO 配置

进入后台管理 → SEO配置，完成以下设置：

| 配置项 | 说明 |
|--------|------|
| 启用 SEO 优化 | 开启 SEO 功能总开关 |
| 网站描述 | 首页 Meta Description，简明扼要描述网站主题 |
| 网站关键词 | 首页 Meta Keywords，用逗号分隔 |
| 默认作者 | 结构化数据中显示的作者名 |
| 默认封面图 | 文章无封面时的默认图片，建议 1200x630 像素 |
| 社交媒体设置 | 配置 Open Graph、Twitter Card |

### 友链配置

进入后台管理 → 资源聚合。

**第一步：修改本站信息**

在资源列表中找到类型为"本站信息"（siteInfo）的记录，点击编辑，修改标题和简介为你的网站名称和描述。这是交换友链时提供给对方的信息。

**第二步：添加友链**

点击"新增资源聚合"，类型选择"友链"。

| 字段 | 说明 |
|------|------|
| 标题 | 对方网站名称 |
| 分类 | 选择"友情链接"或"青出于蓝" |
| 简介 | 对方网站的简短描述 |
| 封面 | 对方网站的头像或 Logo |
| 链接 | 对方网站地址 |

友链可增加网站外链数量，有助于提升搜索引擎信任度。建议与相关领域的博客互换友链。

### 文章 AI 助手配置

进入后台管理 → 文章AI助手，配置全局 AI 模型（API地址、密钥）和智能摘要功能。AI 生成的摘要更有利于 SEO 收录。

### 验证预渲染服务

由于 Vue 是单页应用，搜索引擎爬虫默认只能看到空页面。本项目通过预渲染服务解决这个问题。

**验证方法**：

1. 打开任意文章页面
2. 右键 → 查看页面源代码
3. 确认文章正文内容已存在于 HTML 中

如果只看到 `<div id="app"></div>`，说明预渲染服务未正常工作，请检查 `poetize-prerender` 容器状态。

---

## 第二步：内容储备

**核心原则**：不要空站提交。建议先储备 3-5 篇文章再提交搜索引擎。一定不要一篇文章就提交，这样搜索引擎会认为你的网站没有价值。

### 内容要求

- 每篇文章建议 2000 字以上
- 文章标题包含目标关键词
- 为文章上传封面图（未上传时系统会自动使用默认封面）
- 形成主题群，文章之间相互链接

**示例（技术博客）**：

```
主文章：《2026年Vue3性能优化完全指南》(5000字)
    ├── 《Vite vs Webpack深度对比》
    └── 《Pinia状态管理最佳实践》
```

---

## 第三步：提交搜索引擎

内容储备完成后，主动提交站点地图。

### 确认站点地图

访问 `http(s)://你的域名/sitemap.xml`，确认已包含你的文章。

### 提交至搜索引擎

- [Google Search Console](https://search.google.com/search-console)
- [百度搜索资源平台](https://ziyuan.baidu.com)
- [Bing Webmaster](https://www.bing.com/webmasters)

提交站点地图后，再手动提交最重要的"主文章"URL。

---

## 第四步：持续优化

### 更新频率

每周至少发布 1 篇文章，保持爬虫回访频率。

### 内链建设

新文章发布时，在正文中手动链接到 2-3 篇相关的旧文章。

### 社交分享

文章发布后分享到 Twitter、知乎、微博等平台，引导爬虫发现新内容。

## 白帽 SEO 技巧

以下技巧符合搜索引擎规范，长期有效。

### 标题优化

- **标题长度**：控制在 25-35 个汉字，避免被搜索结果截断
- **关键词前置**：核心关键词放在标题前半部分
- **数字吸引力**：如"10个技巧"、"2026年最新"等增加点击率
- **避免标题党**：标题要与内容相符，否则跳出率高会降低排名

**示例**：
- ❌ `Vue教程` （太笼统）
- ✅ `Vue3组合式API实战：5个提升代码质量的技巧`

### 内容结构

- **标题层级**：文章正文中直接使用 `#` 作为一级章节标题即可，本项目会自动将其降级为 H2（因为 H1 应保留给文章标题）。必要时使用 `##` 作为二级章节标题。建议每篇文章至少包含 3 个一级章节标题
- **首段点题**：前 100 字包含核心关键词，告诉搜索引擎文章主题
- **段落适中**：每段 3-5 行，避免大段文字墙
- **列表和表格**：善用列表和表格提升可读性，搜索引擎也更容易理解

### 关键词布局

- **密度控制**：关键词占全文 2%-3%，过多会被判定为堆砌
- **同义词替换**：使用"Vue"、"Vue.js"、"Vue框架"等变体，避免重复
- **长尾关键词**：除了主关键词，覆盖"Vue3 性能优化方法"等长尾词

### 图片优化

- **Alt 文本**：为每张图片添加描述性 alt 属性
- **图片大小**：控制在 200KB 以内，系统会自动转换为 WebP 格式

### 外链获取（合规方式）

1. **技术社区投稿**：掘金、思否、CSDN 等允许文末放原文链接
2. **GitHub 项目**：在项目 README 中链接相关博客文章
3. **回答问题**：在 Stack Overflow、知乎回答问题时引用自己的深度文章
4. **友链交换**：与同领域博主互换友链

### 用户体验信号

搜索引擎会参考用户行为数据：

| 指标 | 优化方向 |
|------|----------|
| 停留时间 | 增加文章深度、插入代码示例和图表 |
| 跳出率 | 开头直接给出有价值内容，不要废话 |
| 页面速度 | 本项目已优化，但注意图片大小 |
| 移动端体验 | 本项目已适配，确保代码块可横向滚动 |

### 避免的操作

以下操作可能导致降权或被K站：

- ❌ 关键词堆砌（同一关键词大量重复）
- ❌ 隐藏文字（白底白字等）
- ❌ 购买低质量外链
- ❌ 大量采集/洗稿内容
- ❌ 频繁修改已收录文章的标题

### 进阶技巧

**搜索意图匹配**：用户搜索分为信息型（"什么是Vue"）、导航型（"Vue官网"）、交易型（"Vue课程购买"）。技术博客主攻信息型，标题以"如何"、"为什么"、"X种方法"开头更容易命中。

**Featured Snippet 优化**：想抢占搜索结果顶部的精选摘要位置：
- 在文章开头或每个章节开头用 1-2 句话直接回答问题
- 使用编号列表回答"如何做"类问题
- 使用表格对比类内容

**更新旧文章**：
- 每 3-6 个月更新一次高流量文章
- 更新年份标识（如"2025指南"→"2026指南"）
- 补充最新内容，但**不要改标题**（已收录的标题修改会影响排名）

**内链锚文本**：
- 用描述性文字做锚文本，如"参考 [Vue3性能优化指南](/article/123)"
- 避免"点击这里"、"查看更多"等无意义锚文本
- 每篇文章内链 2-5 个，不要过多

**Schema 结构化数据**：本项目已自动生成 Article Schema，如需添加 FAQ Schema，可在文章末尾按以下格式编写：

```markdown
## 常见问题

### 问题1：xxx？

答案内容...

### 问题2：xxx？

答案内容...
```

## 常见问题

### 为什么文章发布后没有被收录？

1. 内容太少，搜索引擎认为价值不高
2. 未主动提交站点地图
3. 网站太新，需要时间建立信任

### 收录后排名很低怎么办？

1. 增加文章深度和字数
2. 建立内链网络
3. 获取外部网站的链接引用
4. 保持更新频率

### 如何查看收录状态？

在搜索引擎中搜索 `site:你的域名`，可查看已收录的页面数量。
