# SEO 优化指南

本指南帮助你的内容更快被搜索引擎收录。

---

## 第一步：部署后配置

部署完成后，在开始写文章之前，先完成以下配置。

### 后台 SEO 配置

进入后台管理 → SEO配置，完成以下设置：

| 配置项 | 说明 |
|--------|------|
| 启用 SEO 优化 | 开启 SEO 功能总开关 |
| 网站描述 | 首页 Meta Description，简明扼要描述网站主题 |
| 网站关键词 | 首页 Meta Keywords，用逗号分隔 |
| 默认作者 | 结构化数据中显示的作者名 |
| 默认封面图 | 文章无封面时的默认图片，建议 1200x630 像素 |
| 社交媒体设置 | 配置 Open Graph、Twitter Card |
| 其它 HTML 标签 | 用于验证搜索引擎所有权的 Meta 标签或其它自定义 HTML |

### 友链配置

进入后台管理 → 资源聚合。

**第一步：修改本站信息**

在资源列表中找到类型为"本站信息"（siteInfo）的记录，点击编辑，修改标题和简介为你的网站名称和描述。这是交换友链时提供给对方的信息。

**第二步：添加友链**

点击"新增资源聚合"，类型选择"友链"。

| 字段 | 说明 |
|------|------|
| 标题 | 对方网站名称 |
| 分类 | 选择"友情链接"或"青出于蓝" |
| 简介 | 对方网站的简短描述 |
| 封面 | 对方网站的头像或 Logo |
| 链接 | 对方网站地址 |

友链可增加网站外链数量，有助于提升搜索引擎信任度。建议与相关领域的博客互换友链。

### 文章 AI 助手配置

进入后台管理 → 文章AI助手，配置全局 AI 模型（API地址、密钥）和智能摘要功能。AI 生成的摘要更有利于 SEO 收录。

### 验证预渲染服务

由于 Vue 是单页应用，搜索引擎爬虫默认只能看到空页面。本项目通过预渲染服务解决这个问题。

**验证方法**：

1. 打开任意文章页面
2. 右键 → 查看页面源代码
3. 确认文章正文内容已存在于 HTML 中

如果只看到 `<div id="app"></div>`，说明预渲染服务未正常工作，请检查 `poetize-prerender` 容器状态。

## 第二步：内容储备

**核心原则**：不要空站提交。建议先储备 3-5 篇文章再提交搜索引擎。一定不要一篇文章就提交，这样搜索引擎会认为你的网站没有价值。

### 内容要求

- 每篇文章建议 2000 字以上
- 文章标题包含目标关键词
- 为文章上传封面图（未上传时系统会自动使用默认封面）
- 形成主题群，文章之间相互链接

**示例（技术博客）**：

```
主文章：《2026年Vue3性能优化完全指南》(5000字)
    ├── 《Vite vs Webpack深度对比》
    └── 《Pinia状态管理最佳实践》
```

## 第三步：提交搜索引擎

内容储备完成后，主动提交站点地图。这能让搜索引擎更快地发现并索引你的新内容。

### 1. 确认站点地图

本项目已自动为你生成站点地图。
访问 `http(s)://你的域名/sitemap.xml`，确认其中已包含你的文章链接。

### 2. 验证站点所有权

在提交之前，你需要证明你是该网站的所有者。本项目对此提供了极其便捷的支持：

1. **HTML 标签（最推荐）**：
   - 在搜索引擎后台获取验证用的 `<meta>` 标签。
   - 进入 **后台管理 → SEO配置**，将标签粘贴至 **搜索引擎验证** 下方的对应输入框中，并点击 **保存配置**。
   - 这样标签会自动插入到首页的 `<head>` 中，无需修改服务器配置或代码。
2. **CNAME 验证**：在你域名的 DNS 解析服务商处（如阿里云、腾讯云、Cloudflare 等），按照搜索引擎的要求添加一条 CNAME 记录。这种方式不依赖服务器，且对单页应用非常友好。

### 3. 各平台提交指南

#### [Google Search Console](https://search.google.com/search-console)
1. **添加资源**：选择"网址前缀"并输入你的博客地址。
2. **验证**：推荐使用 **"HTML 标记"** 方式。
3. **提交 Sitemap**：进入左侧菜单的 **站点地图 (Sitemaps)**，输入 `sitemap.xml` 并提交。

#### [百度搜索资源平台](https://ziyuan.baidu.com)
1. **添加站点**：在"资源收录"中点击"添加网站"。
2. **验证**：完成站点验证后，进入 **资源收录 → 普通收录**。
3. **提交方式**：
   - **sitemap**：在"sitemap"选项卡中提交你的 `sitemap.xml` 完整链接。
   - **API 提交**（推荐）：本项目后端已集成百度准时推送功能，发布文章时会自动推送到百度。

#### [Bing Webmaster Tools](https://www.bing.com/webmasters)
1. **导入**：Bing 支持直接从 Google Search Console 导入站点配置，这是最快捷的方式。
2. **手动添加**：如果选择手动添加，验证方式与 Google 类似。
3. **提交 Sitemap**：在 **站点地图** 页面点击 "提交站点地图"，输入 `sitemap.xml` 完整链接。

> [!TIP]
> 提交站点地图后，建议在各平台的"URL 检查"或"抓取诊断"功能中，手动提交你最重要的"主文章"URL，这样通常能在几分钟到几小时内看到收录效果。

## 第四步：持续优化

### 更新频率

每周至少发布 1 篇文章，保持爬虫回访频率。

### 保证网站稳定性

避免长时间或频繁宕机。搜索引擎爬虫（如 Googlebot）如果多次访问发现网站无法连接（返回 5xx 错误或连接超时），会认为该站不可靠，从而降低权重、减少抓取频率甚至暂时取消收录。

- **减小更新影响**：由于本项目升级或重新部署可能需要较长时间（约 2 小时），建议尽量在低流量时段进行。
- **避免带错运行**：确保部署前的配置（如 SEO 参数、预渲染服务）正确，防止爬虫抓取到错误或空白的页面。

### 内链建设

新文章发布时，在正文中手动链接到 2-3 篇相关的旧文章。

### 社交分享

文章发布后分享到 Twitter、知乎、微博等平台，引导爬虫发现新内容。

## 白帽 SEO 技巧

以下技巧符合搜索引擎规范，长期有效。

### 标题优化

- **标题长度**：控制在 25-35 个汉字，避免被搜索结果截断
- **关键词前置**：核心关键词放在标题前半部分
- **数字吸引力**：如"10个技巧"、"2026年最新"等增加点击率
- **避免标题党**：标题要与内容相符，否则跳出率高会降低排名

**示例**：
- ❌ `Vue教程` （太笼统）
- ✅ `Vue3组合式API实战：5个提升代码质量的技巧`

### 内容结构

- **标题层级**：文章正文中直接使用 `#` 作为一级章节标题即可，本项目会自动将其降级为 H2（因为 H1 应保留给文章标题）。必要时使用 `##` 作为二级章节标题。建议每篇文章至少包含 3 个一级章节标题
- **首段点题**：前 100 字包含核心关键词，告诉搜索引擎文章主题
- **段落适中**：每段 3-5 行，避免大段文字墙
- **列表和表格**：善用列表和表格提升可读性，搜索引擎也更容易理解

### 关键词布局

- **密度控制**：关键词占全文 2%-3%，过多会被判定为堆砌
- **同义词替换**：使用"Vue"、"Vue.js"、"Vue框架"等变体，避免重复
- **长尾关键词**：除了主关键词，覆盖"Vue3 性能优化方法"等长尾词

### 图片优化

- **Alt 文本**：为每张图片添加描述性 alt 属性
- **图片大小**：控制在 200KB 以内，系统会自动转换为 WebP 格式

### 外链获取（合规方式）

1. **技术社区投稿**：掘金、思否、CSDN 等允许文末放原文链接
2. **GitHub 项目**：在项目 README 中链接相关博客文章
3. **回答问题**：在 Stack Overflow、知乎回答问题时引用自己的深度文章
4. **友链交换**：与同领域博主互换友链

### 用户体验信号

搜索引擎会参考用户行为数据：

| 指标 | 优化方向 |
|------|----------|
| 停留时间 | 增加文章深度、插入代码示例和图表 |
| 跳出率 | 开头直接给出有价值内容，不要废话 |
| 页面速度 | 本项目已优化，但注意图片大小 |
| 移动端体验 | 本项目已适配，确保代码块可横向滚动 |

### 避免的操作

以下操作可能导致降权或被K站：

- ❌ 关键词堆砌（同一关键词大量重复）
- ❌ 隐藏文字（白底白字等）
- ❌ 购买低质量外链
- ❌ 大量采集/洗稿内容
- ❌ 频繁修改已收录文章的标题

### 进阶技巧

**搜索意图匹配**：用户搜索分为信息型（"什么是Vue"）、导航型（"Vue官网"）、交易型（"Vue课程购买"）。技术博客主攻信息型，标题以"如何"、"为什么"、"X种方法"开头更容易命中。

**Featured Snippet 优化**：想抢占搜索结果顶部的精选摘要位置：
- 在文章开头或每个章节开头用 1-2 句话直接回答问题
- 使用编号列表回答"如何做"类问题
- 使用表格对比类内容

**更新旧文章**：
- 每 3-6 个月更新一次高流量文章
- 更新年份标识（如"2025指南"→"2026指南"）
- 补充最新内容，但**不要改标题**（已收录的标题修改会影响排名）

**内链锚文本**：
- 用描述性文字做锚文本，如"参考 [Vue3性能优化指南](/article/123)"
- 避免"点击这里"、"查看更多"等无意义锚文本
- 每篇文章内链 2-5 个，不要过多

**Schema 结构化数据**：本项目已自动生成 Article Schema，如需添加 FAQ Schema，可在文章末尾按以下格式编写：

```markdown
## 常见问题

### 问题1：xxx？

答案内容...

### 问题2：xxx？

答案内容...
```

## 关于黑帽 SEO (风险警示)

黑帽 SEO 指的是通过违反搜索引擎服务条款的手段来获取更高排名的技术。虽然这些手段在短期内可能见效极快，但对于个人博客等长期项目来说，其实质是一种**收益并存的高风险博弈**。

### 常见的黑帽 SEO 技巧

-   **伪装 (Cloaking)**：向用户显示的内容与向搜索引擎爬虫显示的内容不同。
-   **站群 (PBN/Site Farms)**：建立大量低质量站点，通过相互链接来人为提高主站权。
-   **桥页 (Doorway Pages)**：专门为填充特定关键词而设计的低质量中间页，旨在将用户重定向到其它页面。
-   **负面 SEO (Negative SEO)**：通过给竞争对手购买大量垃圾外链等手段，试图让对方受到搜索引擎惩罚。
-   **隐藏链接**：在页面中插入用户看不见（如：1像素大小、颜色与背景相同等）的链接。

### 潜在风险

> [!CAUTION]
> 搜索引擎（尤其是 Google）拥有极其强大的 AI 算法（如 SpamBrain）来识别黑帽行为。一旦被发现：
> 1. **全站降权**：网站关键词排名瞬间跌出前 100 名。
> 2. **K 站 (De-indexing)**：网站的所有页面从搜索引擎索引中被彻底抹除。
> 3. **品牌信任度归零**：域名可能被永久列入黑名单，后续即便改邪归正也难以恢复。

对于希望通过文字积累长期价值的博主，**白帽 SEO（即本指南前文所述的方法）是唯一可持续的道路**。

## 常见问题

### 为什么文章发布后没有被收录？

1. 内容太少，搜索引擎认为价值不高
2. 未主动提交站点地图
3. 网站太新，需要时间建立信任

### 收录后排名很低怎么办？

1. 增加文章深度和字数
2. 建立内链网络
3. 获取外部网站的链接引用
4. 保持更新频率

### 如何查看收录状态？

在搜索引擎中搜索 `site:你的域名`，可查看已收录的页面数量。
