# 排障指南

本文档提供 POETIZE 博客系统常见问题的排查和解决方案。

## 项目部署常见问题

### 1. 项目升级时网络中断了

不用慌，可以先把这个项目删掉，部署一个新的博客项目，之后使用`poetize -restore /tmp/poetize_xxx`命令恢复数据，示例操作如下：

```bash
# 假设你当前所在的路径在 awesome-poetize-open 目录内
# 卸载旧项目
poetize -uninstall-all
# 等待卸载完成后
cd ..
# 删除旧项目
rm -rf awesome-poetize-open
# 重新部署
bash <(curl -sL install.leapya.com)
# 等待部署完成后
cd awesome-poetize-open
# 恢复数据（备份文件默认在/tmp目录下，格式为poetize_update_backup_xxx，可用 ls -dt /tmp/poetize_update_backup_* | head -1 查看最新备份）
poetize -restore /tmp/poetize_update_backup_xxx
```

### 2. 申请证书失败

**可能原因：**

1. **域名未解析到服务器**：申请证书前必须确保域名已正确解析到服务器IP。
2. **80端口被占用或未开放**：Certbot 需要使用 80 端口验证域名所有权。
3. **申请次数超限**：Let's Encrypt 有严格的频率限制：
   - **验证失败**：每域名每小时最多 5 次失败（排查问题时极易触发）
   - **重复证书**：同一组域名每周最多 5 次签发（重装系统太频繁会触发）

**解决方案：**

1. 检查域名解析是否生效（使用 `ping 你的域名`）。
2. 确保防火墙已开放 80 端口且未被其他程序占用。
3. 如果次数超限，请等待 1 小时后再试，或先使用 HTTP 模式部署。
4. 手动重新尝试申请：
   ```bash
   docker exec poetize-nginx /enable-https.sh
   ```

### 3. 部署时提示 80/443 端口被占用

**问题表现：** 启动容器时报错 `Bind for 0.0.0.0:80 failed: port is already allocated`。

**解决方案：**

1. **方法一：使用自定义端口 + CDN（推荐）**
   - 部署时直接指定端口（脚本支持自动处理）：
     ```bash
     # 交互模式输入域名时带上端口，如: example.com:8080
     # 或者使用命令行参数: ./poetize install -d example.com:8080
     ```
   - 使用 Cloudflare 等 CDN，在 SSL/TLS 设置中配置回源端口。

2. **方法二：Nginx 反向代理（适合宝塔/1Panel用户）**
   - 如果服务器已安装宝塔或 1Panel 面板：
     1. 部署时指定后端端口（避免端口冲突）：`./poetize install -d example.com:8090`
     2. 在面板中新建网站（绑定域名）。
     3. 添加**反向代理**，目标 URL 填 `http://127.0.0.1:8090`。

## 前端常见问题

### 1. npm install 依赖安装失败

```bash
# 问题：通常是网络问题导致
# 解决：配置淘宝镜像源
npm config set registry https://registry.npmmirror.com

# 然后重新安装
npm install
```

如仍失败，清除缓存重试：

**Linux/macOS:**
```bash
npm cache clean --force
rm -rf node_modules package-lock.json
npm install
```

**Windows PowerShell:**
```powershell
npm cache clean --force
Remove-Item -Recurse -Force node_modules, package-lock.json
npm install
```

### 2. 无法登录后台（点击登录无反应或报错）

**问题原因：**
生产环境（公网 IP 或域名）未开启 HTTPS，导致 **Web Crypto API** 失效，密码加密无法执行。

**解决方案：**
1. 确保服务器已启用 HTTPS（推荐使用 `poetize -update` 重新配置）。
2. 如果仅用于测试，请通过本地回环地址（`http://localhost` 或 `http://127.0.0.1`）访问。
3. 检查浏览器控制台（F12）是否有 `crypto.subtle is undefined` 错误。

### 3. 前端 docker 部署启动后 API 请求失败（跨域/404）

```bash
# 检查后端服务是否正常运行

# 确保后端服务已启动
docker ps | grep poetize-java

# 检查服务内存占用（检查后端服务或者数据库是否因资源不足导致异常）
docker stats
```

### 3. WebSocket 连接失败（聊天室无法连接）

```bash
# WebSocket 服务已集成到 Spring Boot 主程序（端口 8081），路径为 /ws/im
# 本地开发：ws://localhost:8081/ws/im
# 生产环境：通过 Nginx 代理到 wss://你的域名/ws/im

# 确保后端服务已启动（只需检查 8081 端口）
docker ps | grep poetize-java
# Linux:
netstat -tlnp | grep 8081
# macOS:
lsof -i :8081
# Windows PowerShell:
netstat -ano | findstr :8081

# 检查 Nginx WebSocket 代理配置
docker exec poetize-nginx cat /etc/nginx/conf.d/default.conf | grep -A 10 "/ws/im"
```

## Java后端常见问题

### 1. Maven 依赖下载缓慢或失败

检查 `settings.xml` 镜像源配置，已配置华为云镜像，如仍失败可尝试阿里云镜像。

清除 Maven 本地仓库重新下载：

**Linux/macOS:**
```bash
rm -rf ~/.m2/repository
mvn clean install
```

**Windows PowerShell:**
```powershell
Remove-Item -Recurse -Force $env:USERPROFILE\.m2\repository
mvn clean install
```

### 2. Spring Boot 启动失败

```bash
# 检查 JDK 版本（必须是 JDK 25，使用其他版本需修改 pom.xml 配置）
java -version
```

使用 `mvn spring-boot:run` 启动时，错误日志直接打印在控制台。常见错误及解决方案：

**1. 端口被占用**

错误：`Web server failed to start. Port 8081 was already in use`

查看占用进程：
- **Linux/macOS:** `lsof -i :8081`
- **Windows:** `netstat -ano | findstr :8081`

**2. 数据库连接失败**

错误：`Unable to acquire JDBC Connection` 或 `Access denied`

解决：检查 `application.yml` 中数据库配置，确保数据库服务已启动。

**3. Redis 连接失败**

错误：`Unable to connect to Redis`

解决：确保 Redis 服务已启动，检查密码配置。

### 3. 数据库连接密码错误

部署脚本会自动生成随机数据库密码，密码保存在 `.config/db_credentials.txt` 文件中。

```bash
# 查看数据库密码
cat .config/db_credentials.txt

# 使用正确的密码连接数据库
# 假设 ROOT 密码为 abc123xyz（实际以文件中为准）
docker exec -it poetize-mariadb mariadb -uroot -p
# 提示输入密码时，粘贴从文件中获取的 ROOT 密码

# 或使用 poetize 用户连接
docker exec -it poetize-mariadb mariadb -upoetize -p poetize
# 输入文件中的 poetize 用户密码
```

### 4. 数据库初始化失败或表不存在

```bash
# 先获取数据库密码
ROOT_PWD=$(grep "数据库ROOT密码:" .config/db_credentials.txt | cut -d':' -f2 | tr -d ' ')

# 检查是否已执行初始化脚本
docker exec -i poetize-mariadb mariadb -uroot -p${ROOT_PWD} -e "USE poetize; SHOW TABLES;"

# 重新初始化数据库
docker exec -i poetize-mariadb mariadb -uroot -p${ROOT_PWD} poetize < poetize-server/sql/poetry.sql

# 如果 RocksDB 引擎报错，使用兼容性脚本
docker exec -i poetize-mariadb mariadb -uroot -p${ROOT_PWD} poetize < poetize-server/sql/poetry_old.sql
```

### 5. 本地开发环境数据库连接失败

如果你是全新安装的 MariaDB/MySQL 和 Redis，请确保密码配置正确：

**问题表现**：启动后端时报 `Access denied` 或 `Authentication failed`

**解决方案**：

1. **检查数据库密码**：开发环境默认使用密码 `123456`
   
   ```bash
   # 测试数据库连接（mysql 或 mariadb 命令均可）
   mysql -uroot -p123456
   # 或
   mariadb -uroot -p123456
   ```

2. **如果密码不是 123456**，需要修改配置文件：
   
   编辑 `poetize-server/poetry-web/src/main/resources/application.yml`：
   ```yaml
   spring:
     datasource:
       password: 你的实际密码
   ```

3. **Redis 密码配置**：
   
   如果 Redis 设置了密码，编辑 `application.yml`：
   ```yaml
   spring:
     redis:
       password: 你的Redis密码
   ```

4. **重置 MariaDB 密码为 123456**：
   
   ```bash
   # Linux/macOS（使用 mariadb 或 mysql 命令）
   sudo mariadb -e "ALTER USER 'root'@'localhost' IDENTIFIED BY '123456'; FLUSH PRIVILEGES;"
   
   # Windows（以管理员身份运行 CMD）
   mysql -u root -e "ALTER USER 'root'@'localhost' IDENTIFIED BY '123456'; FLUSH PRIVILEGES;"
   ```


## Docker 环境问题

### 1. Docker 容器无法启动

```bash
# 查看容器状态
docker ps -a

# 查看容器日志
docker logs poetize-java
docker logs poetize-mariadb

# 重启特定容器
docker restart poetize-java
```

### 2. 容器启动后立即退出

```bash
# 查看退出原因
docker logs --tail 50 容器名

# 检查资源限制（内存不足）
docker stats

# 检查配置文件语法
docker compose config
```

### 3. 数据库容器健康检查失败

```bash
# 获取数据库密码
ROOT_PWD=$(grep "数据库ROOT密码:" .config/db_credentials.txt | cut -d':' -f2 | tr -d ' ')

# 手动检查数据库连接
docker exec -it poetize-mariadb mariadb -uroot -p${ROOT_PWD}

# 检查存储引擎插件
docker exec -it poetize-mariadb mariadb -uroot -p${ROOT_PWD} -e "SHOW ENGINES;"

# 查看数据库错误日志
docker logs poetize-mariadb
```

## 网络与访问问题

### 1. 无法访问后台管理页面

```bash
# 检查 Nginx 配置
docker exec poetize-nginx nginx -t

# 重启 Nginx
docker restart poetize-nginx

# 检查路由配置
# 后台地址: http://域名/admin
```

### 2. 静态资源 404

```bash
# 检查前端构建产物
ls poetize-web/dist/

# 检查 Nginx 静态文件映射
docker exec poetize-nginx cat /etc/nginx/conf.d/default.conf | grep "location"
```

### 3. HTTPS 证书问题

```bash
# 重新申请证书
docker exec poetize-nginx /enable-https.sh

# 检查证书有效期
docker exec poetize-nginx certbot certificates

# 查看证书续期日志
docker logs poetize-nginx | grep certbot
```

## 性能与调试

### 1. 接口响应慢

```bash
# 检查 Redis 缓存状态
docker exec -it poetize-redis redis-cli
> INFO stats
> DBSIZE

# 查看 Java 应用 JVM 状态
docker exec poetize-java jstack 1

# 检查数据库慢查询
ROOT_PWD=$(grep "数据库ROOT密码:" .config/db_credentials.txt | cut -d':' -f2 | tr -d ' ')
docker exec -it poetize-mariadb mariadb -uroot -p${ROOT_PWD} -e "SHOW FULL PROCESSLIST;"
```

### 2. 查看实时日志

```bash
# 查看所有服务日志
docker compose logs -f

# 查看特定服务日志
docker logs -f poetize-java

# 查看最近 100 行日志
docker logs --tail 100 poetize-java
```

### 3. 内存占用过高

```bash
# 查看容器资源使用
docker stats

# 调整 Java 堆内存与容器上限（修改 .env 或 docker-compose.yml）
JAVA_OPTS=-Xms512m -Xmx1g
JAVA_MEMORY_LIMIT=1536m

# 重启服务生效
docker compose restart poetize-java
```
