## 快速入门

> 以下步骤可在 5 分钟内完成一个最小可用插件。

**第一步 — 创建目录和 manifest**

```
my-first-plugin/
├── manifest.json
└── frontend/
    └── index.js
```

```json
{
  "name": "my-first-plugin",
  "displayName": "我的第一个插件",
  "version": "1.0.0",
  "author": "你的名字",
  "pluginType": "widget",
  "description": "在文章底部追加一段自定义内容"
}
```

只有三个字段是必填的：`name`、`version`、`pluginType`。

**第二步 — 编写前端逻辑**

```js
// frontend/index.js
PoetizePlugin.on('onArticleRender', function (ctx) {
  if (document.getElementById('my-banner')) return

  var el = document.createElement('div')
  el.id = 'my-banner'
  el.style.cssText = 'margin:20px 0;padding:12px 16px;background:#f0f4ff;border-radius:8px;border-left:4px solid #4f8ef7'
  el.textContent = '这是我的第一个插件！'

  if (ctx.element) ctx.element.appendChild(el)
})
```

**第三步 — 打包为 .zip**

```bash
# Windows PowerShell
Compress-Archive -Path my-first-plugin\* -DestinationPath my-first-plugin-1.0.0.zip

# Linux / macOS
cd my-first-plugin && zip -r ../my-first-plugin-1.0.0.zip .
```

**第四步 — 安装并验证**

1. 登录后台管理 → 插件管理
2. 将 `.zip` 文件拖入上传区，或点击「安装插件」按钮选择文件
3. 预览 manifest 信息后点击「确认安装」
4. 刷新博客页面，打开任意一篇文章查看底部效果

## 插件包规范

### 目录结构

```
plugin-name/
├── manifest.json            必须 — 插件元数据
├── frontend/                可选 — 前端资源
│   ├── index.js               主入口 JS
│   └── style.css              样式文件
├── backend/                 可选 — 后端资源
│   └── main.groovy            Groovy 脚本主入口
└── sql/                     可选 — 数据库迁移
    ├── install.sql              安装时执行
    ├── uninstall.sql            卸载时执行
    └── upgrade.sql              升级时执行
```

### 约束条件

| 项目       | 要求                                             |
|:-----------|:-------------------------------------------------|
| 文件格式   | `.zip`                                           |
| 最大体积   | 10 MB                                            |
| 文件编码   | UTF-8                                            |
| 路径匹配   | 必须严格匹配上述路径名（大小写敏感）             |
| 必须包含   | `manifest.json`（位于 zip 根目录）               |

## manifest.json 规范

### 最小示例

```json
{
  "name": "word-count",
  "version": "1.0.0",
  "pluginType": "widget"
}
```

### 完整示例

```json
{
  "name": "word-count",
  "displayName": "字数统计",
  "version": "1.2.0",
  "author": "张三",
  "pluginType": "article_widget",
  "description": "在文章末尾显示字数和预计阅读时间",
  "minAppVersion": "3.0.0",
  "permissions": ["dom"],
  "hooks": {
    "frontend": ["onArticleRender"],
    "backend": ["onArticleSave"]
  },
  "configSchema": {
    "wordsPerMinute": {
      "type": "number",
      "label": "每分钟阅读字数",
      "defaultValue": 300,
      "description": "用于计算预计阅读时间"
    },
    "showBadge": {
      "type": "boolean",
      "label": "显示徽标",
      "defaultValue": true
    }
  }
}
```

### 字段说明

| 字段            | 类型     | 必填 | 说明                                                       |
|:----------------|:---------|:----:|:-----------------------------------------------------------|
| `name`          | string   | 是   | 唯一标识。小写字母 + 数字 + 连字符，以字母开头。如 `word-count` |
| `version`       | string   | 是   | 语义化版本号。如 `1.0.0`                                   |
| `pluginType`    | string   | 是   | 插件类型，见下方表格                                       |
| `displayName`   | string   |      | 显示名称，支持中文                                         |
| `author`        | string   |      | 作者名                                                     |
| `description`   | string   |      | 一句话描述                                                 |
| `minAppVersion` | string   |      | 最低兼容 Poetize 版本（预留字段）                          |
| `permissions`   | string[] |      | 所需权限列表（预留字段）                                   |
| `hooks`         | object   |      | 声明使用的钩子（文档用途，不影响运行）                     |
| `configSchema`  | object   |      | 配置项定义，详见「配置系统」章节                           |

### 插件类型

| pluginType         | 适用场景     | 示例                 |
|:-------------------|:-------------|:---------------------|
| `widget`           | 通用小组件   | 时钟、天气、打赏按钮 |
| `article_widget`   | 文章增强     | 字数统计、阅读进度条 |
| `sidebar_widget`   | 侧边栏组件   | 标签云、热门文章     |
| `comment_widget`   | 评论增强     | 表情包、@提及        |
| `theme`            | 主题/样式    | 暗色增强、节日主题   |
| `analytics`        | 统计分析     | 阅读热力图           |
| `notification`     | 通知推送     | 评论通知             |

> 也可以使用任意自定义字符串作为 pluginType。

## 前端开发

### 运行环境

- 插件 JS 代码在用户浏览器中通过 `new Function()` 执行
- 全局对象 `window.PoetizePlugin` 始终可用
- 插件 CSS 通过 `<style>` 标签注入 `<head>`
- 插件加载错误不会阻塞主页面渲染

### 插件 SDK — PoetizePlugin

#### 注册钩子

```js
PoetizePlugin.on('onArticleRender', function (ctx) {
  console.log('文章已渲染:', ctx.title)
  console.log('DOM 容器:', ctx.element)
})
```

#### 获取插件配置

```js
// 参数：插件 key、配置项名、默认值
var color = PoetizePlugin.api.getConfig('my-plugin', 'themeColor', '#333')
```

#### 获取当前文章

```js
var article = PoetizePlugin.api.getArticle()
// 非文章页返回 null
```

#### 获取当前登录用户

```js
var user = PoetizePlugin.api.getUser()
// 未登录返回 null
```

#### Toast 提示

```js
PoetizePlugin.api.showToast('操作成功', 'success')
// 可选 type: 'success' | 'warning' | 'error' | 'info'
```

#### 插入 HTML

```js
var target = document.querySelector('.entry-content')
PoetizePlugin.api.insertHtml(target, 'beforeend', '<div>自定义内容</div>')
// position: 'beforebegin' | 'afterbegin' | 'beforeend' | 'afterend'
```

#### 注册侧边栏组件

```js
var mount = PoetizePlugin.api.registerSidebarWidget('my-clock', '实时时钟')
if (mount) {
  mount.innerHTML = '<div id="clock">00:00:00</div>'
}
```

### 编码建议

```js
// 建议：使用 IIFE 包裹，防止全局变量污染
;(function () {
  'use strict'

  // 建议：第一行检查防重，避免热更新时重复注入
  if (document.getElementById('my-plugin-root')) return

  PoetizePlugin.on('onArticleRender', function (ctx) {
    // ...
  })
})()
```

- 使用 `var` 而非 `let/const`，兼容旧浏览器
- 为注入的 DOM 元素设置唯一 `id`，在函数开头检查是否已存在
- 耗时操作使用 `setTimeout` 避免阻塞渲染

## 后端开发

### 运行环境

| 项目       | 说明                                  |
|:-----------|:--------------------------------------|
| 语言       | Apache Groovy 4.0                     |
| 执行环境   | Spring Boot 内的 GroovyClassLoader    |
| 可用变量   | `log`（SLF4J Logger）                 |
| 错误隔离   | 异常被捕获，不影响主业务流程          |

### 钩子函数

在 `backend/main.groovy` 中定义与钩子同名的函数，系统自动注册：

```groovy
def onArticleSave(ctx) {
    log.info("文章已保存: ID={}, 标题={}", ctx.articleId, ctx.title)
}

def onCommentPublish(ctx) {
    log.info("新评论: {}", ctx.contentSnippet)
}

def onUserRegister(ctx) {
    log.info("新用户: {} (ID: {})", ctx.username, ctx.userId)
}
```

### 注意事项

- 钩子函数在主线程内同步执行，请勿执行耗时操作（网络请求、大文件 I/O）
- 日志通过 `log.info()` / `log.warn()` / `log.error()` 输出到 Spring Boot 控制台
- 若需调试已加载的后端插件列表，访问 `GET /admin/plugin/loaded-backends`

## 钩子 API 参考

### 前端钩子

| 钩子名              | 触发时机                   | ctx 字段                                                                |
|:---------------------|:---------------------------|:------------------------------------------------------------------------|
| `onArticleRender`    | 文章内容 DOM 渲染完成后    | `articleId` (Number)、`title` (String)、`element` (HTMLElement)         |

> `element` 是 `.entry-content` 容器节点，可直接操作其子元素。

### 后端钩子

| 钩子名              | 触发时机           | ctx 字段                                                                 |
|:---------------------|:-------------------|:-------------------------------------------------------------------------|
| `onArticleSave`      | 文章保存成功后     | `articleId` (Long)、`title` (String)、`userId` (Long)                   |
| `onCommentPublish`   | 评论入库成功后     | `commentId` (Long)、`articleId` (Long/null)、`userId` (Long)、`contentSnippet` (String) |
| `onUserRegister`     | 新用户注册成功后   | `userId` (Long)、`username` (String)                                    |

> `contentSnippet` 最长 200 字符，超出部分以 `...` 截断。
> `articleId` 在留言板评论时为 `null`。

### 触发链路

```
用户操作
  → 业务 Service（ArticleServiceImpl / CommentServiceImpl / UserServiceImpl）
    → PluginHookManager                ← 安全包装，异常不影响主流程
      → GroovyPluginEngine             ← 遍历所有已加载插件
        → 各插件的 def onXxx(ctx)      ← 依次执行
```

{
  "configSchema": {

      "type": "number",

### 支持的字段类型

| type      | 控件       | 示例值       |
|:----------|:-----------|:-------------|

var fontSize = PoetizePlugin.api.getConfig('my-plugin', 'fontSize', 14)

## 数据库迁移

### install.sql — 安装时执行

```sql
CREATE TABLE IF NOT EXISTS plugin_my_widget_stats (
    id          INT          NOT NULL AUTO_INCREMENT,
    article_id  BIGINT       NOT NULL,
    view_count  INT          DEFAULT 0,
    created_at  DATETIME     NOT NULL DEFAULT NOW(),
    PRIMARY KEY (id),
    INDEX idx_article (article_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4;
```

### uninstall.sql — 卸载时执行

```sql
DROP TABLE IF EXISTS plugin_my_widget_stats;
```

### upgrade.sql — 升级时执行

```sql
ALTER TABLE plugin_my_widget_stats ADD COLUMN updated_at DATETIME DEFAULT NOW();
```

> 升级 SQL 仅在版本号升高时触发，已执行过的版本不会重复执行。

### 命名规范

所有表名**必须**以 `plugin_` 前缀开头。推荐格式：

```
plugin_{插件name}_{业务表名}
```

例如插件 `word-count` 的统计表：`plugin_word_count_stats`。

## 安全策略与限制

### SQL 安全校验

系统通过 `SqlSafetyValidator` 自动校验插件提交的 SQL 脚本：

**允许的操作：**

| 操作                 | 条件                       |
|:---------------------|:---------------------------|
| `CREATE TABLE`       | 表名以 `plugin_` 开头     |
| `ALTER TABLE`        | 表名以 `plugin_` 开头     |
| `INSERT / UPDATE / DELETE` | 表名以 `plugin_` 开头 |
| `SELECT`             | 无限制                     |
| `DROP TABLE`         | 仅在卸载 SQL 中允许       |

**禁止的操作：**

- 操作非 `plugin_` 前缀的任何表（如 `user`、`article`）
- `DROP DATABASE`、`GRANT`、`REVOKE`、`SHUTDOWN`
- `LOAD DATA`、`INTO OUTFILE`、`INTO DUMPFILE`
- 访问 `INFORMATION_SCHEMA`、`MYSQL.*`、`SYS.*`、`PERFORMANCE_SCHEMA`
- 安装 SQL 中使用 `DROP TABLE`

### Groovy 脚本安全

- 脚本在 `GroovyClassLoader` 沙箱中编译运行
- 钩子执行异常由 `PluginHookManager` 捕获，不影响主业务流程
- 可用变量仅有 `log`（SLF4J Logger）

### 前端安全

- 插件 JS 通过 `new Function()` 执行，与主应用存在一定隔离
- 加载或执行失败仅输出 `console.error`，不阻塞页面

### 系统插件保护

标记为 `isSystem = true` 的内置插件（鼠标特效、看板娘等）不允许通过接口卸载。

## 支付插件开发

支付插件是 `pluginType = "payment"` 的特殊插件，通过 `.zip` 包安装后即可作为支付平台使用，实现文章付费和全站会员功能，无需修改或重新编译核心代码。

### 工作原理

```
安装的 Groovy 支付插件
         ↓
  GroovyPaymentAdapter      ← Java 桥接器，实现 PaymentProvider 接口
         ↓
  PaymentService            ← 优先匹配 Java Provider，找不到时回退到 Groovy
         ↓
  article_payment 表        ← 支付记录由 PaymentService 统一写入，与插件无关
```

### 支付插件约定方法

`backend/main.groovy` 必须实现以下四个函数，否则对应功能将返回 `null` 或抛出异常：

| 函数名 | 调用时机 | 必须返回 |
|:-------|:---------|:---------|
| `getPaymentUrl(ctx)` | 用户点击「付费解锁」 | 支付链接 URL 字符串（非空） |
| `verifyCallback(ctx)` | 支付平台发送 Webhook 回调 | Map（见下方格式） |
| `testConnection(ctx)` | 管理员点击「测试连接」 | Boolean |
| `getSuccessResponse(ctx)` | 回调验证成功后 | 返回给支付平台的响应字符串 |

#### getPaymentUrl 的 ctx

| 字段 | 类型 | 说明 |
|:-----|:-----|:-----|
| `ctx.articleId` | Integer | 文章 ID；为 `0` 时表示购买全站会员 |
| `ctx.userId` | Integer | 博客用户 ID |
| `ctx.config` | Map | 插件配置参数（来自 manifest.json configSchema） |

#### verifyCallback 的 ctx

| 字段 | 类型 | 说明 |
|:-----|:-----|:-----|
| `ctx.params` | `Map<String,String>` | 支付平台 POST 的所有请求参数 |
| `ctx.config` | Map | 插件配置参数 |

#### verifyCallback 的返回 Map

| key | 类型 | 说明 |
|:----|:-----|:-----|
| `verified` | Boolean | **必须**，验签是否成功 |
| `platformOrderId` | String | 平台订单号 |
| `customOrderId` | String | 自定义订单号（格式 `u{userId}_a{articleId}` 时系统自动解析） |
| `amount` | BigDecimal/Number/String | 支付金额 |
| `parsedUserId` | Integer | 博客用户 ID（也可通过 customOrderId 自动解析） |
| `parsedArticleId` | Integer | 文章 ID（也可通过 customOrderId 自动解析） |
| `remark` | String | 备注（可选） |

### 最小支付插件示例

```groovy
// backend/main.groovy — 最小支付插件骨架

def getPaymentUrl(ctx) {
    def config    = ctx.config
    def articleId = ctx.articleId
    def userId    = ctx.userId
    def orderId   = "u${userId}_a${articleId}_${System.currentTimeMillis()}"

    // 构造支付链接（替换为实际支付平台 API）
    return "https://pay.example.com/pay?order=${orderId}&amount=5.00"
}

def verifyCallback(ctx) {
    def params = ctx.params

    // 验签（示例：直接信任，实际应校验签名）
    def outTradeNo = params.get("out_trade_no")
    def tradeNo    = params.get("trade_no")
    def money      = params.get("money") ?: "0"

    return [
        verified:        true,
        platformOrderId: tradeNo,
        customOrderId:   outTradeNo,   // 格式 u42_a15 时系统自动解析用户和文章 ID
        amount:          new BigDecimal(money)
    ]
}

def testConnection(ctx) {
    return ctx.config.get("apiUrl") != null
}

def getSuccessResponse(ctx) {
    return "success"
}
```

### Webhook 回调 URL

在支付平台后台配置的回调地址格式为：

```
https://yourdomain.com/payment/webhook/{插件的 name 字段}
```

例如插件 `name` 为 `my-pay-plugin`，回调 URL 为：
```
https://yourdomain.com/payment/webhook/my-pay-plugin
```

### 完整参考实现

`poetize-server/plugin-examples/alipay-f2f-plugin/` 包含了完整的支付宝当面付 Groovy 实现，含 RSA2 签名、验签、HTTP 调用支付宝网关等完整流程，可直接作为开发新支付插件的模板。


### 安装后没有效果

1. 确认插件在后台管理已启用
2. 清除浏览器缓存后刷新页面
3. 打开浏览器控制台（F12），搜索 `[PoetizePlugin]` 查看是否有错误日志

### manifest.json 校验报错

- `name` 只能包含小写字母、数字和连字符，且以字母开头
- `version` 和 `pluginType` 为必填项

### SQL 执行报错

- 表名必须以 `plugin_` 前缀开头
- 安装 SQL 中不能包含 `DROP TABLE`（仅卸载 SQL 可用）
- 请确认 SQL 语法兼容 MariaDB / MySQL

### 如何调试 Groovy 脚本

- 使用 `log.info()` 等方法输出日志，查看 Spring Boot 控制台即可
- 调用 `GET /admin/plugin/loaded-backends` 确认脚本是否已加载

### 插件能否读取业务表

不能。SQL 安全校验器限制插件只能操作 `plugin_` 前缀的表。如需文章、用户等数据，请通过钩子的 `ctx` 上下文获取。
